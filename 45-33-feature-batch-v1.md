> Этот файл — задачи для следующей итерации разработки. **Только то, чего
> сейчас нет в проекте.** Технические доделки текущего кода (расширение базы
> фактов, рандомизатор v3, баги вроде экспорта-без-обработчика, юбилейные

## Общие правила для всех задач

1. **Использовать токены из `AppTheme`** (`Views/Components/Theme.swift`).
   Не хардкодить HEX и шрифты.
2. **Тексты — через `VoiceContent`** (`Services/VoiceContent.swift`).
   Если нужного ключа нет — добавить в общий файл, не разбрасывать строки
   по вьюхам.
3. **Деструктивные действия — всегда через модалку подтверждения.**
4. **Не вводить новых внешних зависимостей** (Apple SDK only — это
   зафиксировано в README-IOS.md).
5. **Все новые сущности — Codable**, чтобы будущий экспорт/импорт JSON
   подхватил их без переделки моделей.
6. **Текст — на русском, нижний регистр заголовков** где это уже принято
   в коде (стиль «винил-стор с другом»).

## 1. Сортировка и фильтрация полки

**Что сейчас:** `ShelfView` показывает все пластинки в порядке `addedAt
descending` (см. `@Query(sort: \VinylRecord.addedAt, order: .reverse)`). У
`ShelfView` уже есть `@State sortOrder: ShelfSortOrder` — но переключателя нет.

**Что добавить:**
- Меню сортировки в toolbar полки (`ToolbarItem(.topBarTrailing)` рядом с
  иконкой карты).
- Варианты сортировки: «недавние», «давние», «А-Я (артист)», «Я-А (артист)»,
  «дороже», «дешевле», «по году ↓», «по году ↑».
- Поле поиска по тексту (артист + название), сворачивается в иконку 🔍
  в неактивном состоянии. Поиск работает локально по `records`, не делает
  запросов к Discogs.
- Сохранять выбранную сортировку в `@AppStorage("shelfSortOrder")`.

**Файлы:** `Views/ShelfView.swift`, `Models/VinylRecord.swift` (расширить
`ShelfSortOrder` если ещё неполный).

**Критерии приёмки:**
- Меню сортировки открывается одним тапом, видны все 8 вариантов.
- Поиск фильтрует по `artist.localizedCaseInsensitiveContains(query) ||
  title.localizedCaseInsensitiveContains(query)`.
- Выбранная сортировка переживает перезапуск приложения.
- Пустой результат поиска показывает `VoiceContent.phrase(.emptySearchResults)`.

## 2. Заметка к пластинке + дата приобретения

**Что сейчас:** в `VinylRecord` есть `story: String` (приватная история) и
`addedAt: Date` (когда добавлено в приложение). Нет даты, **когда пластинка
была куплена в реальности**.

**Что добавить:**
- Поле `purchasedAt: Date?` в модель `VinylRecord` (optional — чтобы старые
  записи остались валидными).
- Поле `purchaseLocation: String` (где куплено — текстовое поле, опционально).
- Поля доступны в `EditRecordView` и в `RecordDetailView`.
- На карточке пластинки в детальном виде показывать `purchasedAt` форматом
  «октябрь 2024» (не точная дата — лишняя точность), если поле заполнено.

**Файлы:** `Models/VinylRecord.swift`, `Views/EditRecordView.swift`,
`Views/RecordDetailView.swift`.

**Критерии приёмки:**
- Миграция SwiftData проходит чисто — старые записи получают `nil` в новых
  полях.
- При редактировании поля сохраняются в SwiftData.
- На детальном виде поля показываются только если заполнены (пустые — не
  отображаются, как и остальные пустые поля).

## 3. Множественный выбор и пакетные действия на полке

**Что сейчас:** действия с пластинкой только через её детальный вид. Удалить
20 пластинок одним движением нельзя.

**Что добавить:**
- В `ShelfView` — режим «выбрать несколько» (через долгий тап на пластинке
  или кнопку «Выбрать» в toolbar).
- В режиме выбора: чекбоксы на строках, нижняя панель действий: «Удалить N»,
  «Добавить тег…», «Перенести в вишлист».
- Подтверждение через alert для удаления (`VoiceContent.phrase(.warnDuplicate)`
  не подходит — нужен новый ключ `.confirmBulkDelete` с плейсхолдером `{N}`).

**Файлы:** `Views/ShelfView.swift`, `Services/VoiceContent.swift`
(новый ключ).

**Критерии приёмки:**
- Долгий тап включает режим выбора, кнопка «Готово» возвращает в обычный.
- В режиме выбора FAB и фильтры скрываются.
- Удаление работает на N записей одним вызовом `modelContext.delete()` + save.
- Действие «Добавить тег» открывает sheet с полем ввода тега, применяет ко
  всем выбранным.

## 4. Импорт пластинок из CSV / Discogs export

**Что сейчас:** добавить можно только по одной или через сканер. Если
коллекционер уже ведёт коллекцию в Discogs/Excel — переносить руками

- Использовать `.fileImporter` для выбора файла.

  без сторонних библиотек).
- Перед импортом — превью: «найдено 234 строки, импортировать?».

**Файлы:** `Views/ProfileView.swift`, новый

**Критерии приёмки:**

- Импорт работает в `Task.detached` — UI не блокируется.
- Дубликаты (по artist + title + year) не импортируются повторно,
  считаются как «пропущено».
- В конце — отчёт: «Импортировано: 198, пропущено как дубликаты: 36».

## 5. Резервная копия в файл (.json)

**Что сейчас:** в `ProfileView` есть пункт «Экспорт коллекции (JSON)» без
обработчика. Это упомянуто в баг-листе как доделка, **но реализация формата
бэкапа и его восстановления — отдельная задача**, потому что бэкап должен
включать всё, не только пластинки.

**Что добавить:**
- Сервис `BackupService` с двумя методами: `export() -> Data` и
  `import(data: Data) throws`.
- Формат бэкапа — JSON со всем содержимым:
  ```json
  {
    "version": 1,
    "exportedAt": "2026-05-30T...",
    "records": [...],
    "wishlist": [...],
    "collections": [...],
    "profile": {...},
    "userStores": [...]
  }
  ```
- Кнопка «Создать резервную копию» в профиле → `.fileExporter`.
- Кнопка «Восстановить из копии» → `.fileImporter` → подтверждение
  «Это заменит текущие данные. Продолжить?» → восстановление.
- Фото пластинок включать в бэкап как base64 (если фото большое — это
  раздует файл; ввести опцию «без фото» как чекбокс на экспорте).

**Файлы:** новый `Services/BackupService.swift`,
`Views/ProfileView.swift`.

**Критерии приёмки:**
- Экспорт даёт валидный JSON, который читается любым редактором.
- Импорт восстанавливает коллекцию идентично — все теги, фото, истории.
- При попытке импорта поверх непустой коллекции — обязательное
  подтверждение.
- Версионирование (`"version": 1`) — на случай изменения схемы в будущем.

## 6. Лоадер при скачивании обложки с Discogs

**Что сейчас:** в `AddSingleView` обложка скачивается синхронно с
сохранением пластинки (если `shouldDownloadCover`). Если интернет медленный,
пользователь видит зависший UI без объяснения.

**Что добавить:**
- В `AddSingleView` и `AddBatchView` при сохранении показывать промежуточное
  состояние «Сохранение…» / «Тянем обложку…».
- Скачивание обложки — в фоне, **не блокирует** сохранение карточки.
  Пластинка сохраняется сразу с цветом-fallback, обложка догружается
  потом, при готовности — обновляет `photoData` записи.
- Если скачивание обложки упало (нет сети, 404) — пластинка остаётся с
  цветовой обложкой, ошибка не показывается пользователю (не критично).

**Файлы:** `Views/AddSingleView.swift`, `Views/AddBatchView.swift`,
`Services/Discogs/DiscogsService.swift`.

**Критерии приёмки:**
- После тапа «Добавить на полку» — пластинка появляется на полке
  максимум за 200мс (без ожидания обложки).
- Обложка догружается в течение нескольких секунд и появляется на
  существующей записи без перезапуска.
- При отсутствии интернета пластинка всё равно сохраняется.

## 14. iCloud Sync через CloudKit

**Что сейчас:** SwiftData локальный. В README упомянут как «следующий шаг»,
но в задачи это не выносилось. **Это серьёзная задача, требует Apple
Developer аккаунт и iCloud-контейнер.**

**Что добавить:**
- В `CrateVinylApp.init` — изменить `ModelConfiguration` на
  `.init(schema: schema, isStoredInMemoryOnly: false,
  cloudKitDatabase: .private(.init(containerIdentifier: "iCloud.crate.45-33")))`.
- В `Resources/Crate.entitlements` — добавить `com.apple.developer.icloud-container-identifiers`
  и `com.apple.developer.icloud-services` (CloudKit).
- Все `@Model` классы должны иметь дефолты для всех полей (требование
  CloudKit). Проверить `VinylRecord`, `WishlistEntry`, `SavedCollection`,
  `UserProfile`, `UserVinylStore` — добавить дефолты, где их нет.
- В профиле — переключатель «Синхронизация iCloud» с состоянием
  «включено / выключено / ошибка».

**Файлы:** `App/CrateVinylApp.swift`, `Resources/Crate.entitlements`,
все `Models/*.swift`, `Views/ProfileView.swift`.

**Критерии приёмки:**
- На устройстве, подключённом к Apple ID, коллекция автоматически
  синхронизируется на iPad/Mac с тем же ID.
- Удаление пластинки на одном устройстве удаляет её на другом в течение
  ~1 минуты.
- Если iCloud отключён в системе — приложение работает локально без
  ошибок.
- Поле `photoData` (помечено `.externalStorage`) корректно
  синхронизируется через CloudKit Assets.

> **Зависимость:** требуется обновлённый `DISCOGS_TOKEN` (см. баг-лист),
> Apple Developer Program ($99/год) и созданный iCloud-контейнер в
> developer.apple.com.
