# Концепт приложения: каталог коллекции винила
### Чекпойнт брейнсторма — версия 4

> Изменения относительно v3: зафиксировано дизайн-направление («Аналоговый
> стол»), проработан визуальный язык и набор экранов, принято навигационное
> решение (профиль и карта переезжают в верхнюю часть главного экрана).
> Содержательная часть концепта (разделы 1–9, 11–14) — без изменений с v3.

---

## 1. Суть проекта

Хобби-проект: мобильное приложение для **коллекционеров винила**, с потенциалом
развития в универсального коллекционера (фигурки, антиквариат и т.д.).

Цель — простое, виральное приложение, закрывающее реальные боли ниши. Первичная
задача — «взлёт». Монетизация — ожидания умеренные, план масштабируется.

**Почему эта ниша:** автор сам коллекционирует винил, фигурки, антиквариат —
знает боли изнутри, может быть первым пользователем. Коллекционеры — редкое
сочетание виральной и платёжеспособной аудитории.

---

## 2. Рамки и ограничения

- **Платформа:** мобильные сторы (App Store + Google Play). Автор в Сербии.
- **Разработка:** максимум на ИИ. Есть part-time разработчик как опция.
  MVP — с минимумом рисков.
- **Ключевой принцип:** чем меньше бэкенда, тем больше вытянет ИИ. MVP —
  максимально на устройстве (офлайн).
- **Тон продукта:** мягкий «вайб», лёгкий гиковский стёб — «винил-стор с другом».

---

## 3. Сквозные продуктовые принципы

- **Теги вместо папок.** «Коллекция» = сохранённый фильтр по тегу.
- **Discogs как помощник при добавлении, не как движок.** Обращение к API
  только в момент поиска; данные сохраняются локально.
- **Хорошее умолчание + опциональная настройка.** Умолчание настолько хорошее,
  что большинство им и остаётся. Настройка — для меньшинства, не на пути у
  большинства. Настройки запоминаются как стиль пользователя.
- **Вайб дозирован.** Стёб — на редких/эмоциональных моментах. На рутине и
  конвейере — тон спокойный.
- **Бесплатная версия честно полезна и самодостаточна.** Платно — только то, без
  чего приложение всё равно отличное.

---

## 4. Факты по Discogs API

- API публичный и бесплатный, платных тарифов нет.
- Лимиты: 60 запросов/мин (аутентифицированно), 25/мин (без аутентификации).
- API НЕ отдаёт цены → рыночный PnL невозможен. Скрейпинг запрещён Terms of Use.
- Лимит по ключу приложения — при росте пользователи делят общий лимит.
- По одному штрихкоду может вернуться несколько кандидатов-переизданий.

---

## 5. Экран добавления пластинки

Одна точка входа — «Добавить пластинку», внутри тумблер режима.

**Режим «по одной» (размеренный):** спокойный поток, здесь живёт вайб (факт о
релизе). Сценарий — новое приобретение.

**Режим «списком» (конвейер):** массовый занос. Серийное сканирование штрихкодов
→ список-черновик (открыть карточку, редактировать, удалить, доввести вручную).
Неоднозначные совпадения — флаг «проверь меня». Дубликаты — мягкое
предупреждение, не блокировка. Вайб на конвейере выключен.

Текстовый поиск — основной путь; штрихкод — необязательный шорткат.

---

## 6. Витрина

Витрина = вирусный двигатель, «продукт внутри продукта».

**Формат:** MVP — картинка. Витрина-страница — на будущее, с бэкендом.

**Карточка пластинки:** умный шаблон — фото пользователя в центре, откат на
стоковую обложку без фото. Артист/альбом/год + «понтовые» поля + опционально
история. Цена покупки на витрину не выносится.

**Карточка коллекции:** MVP — «полка» и «сетка». «Постер-сводка» — позже.

**Генерация:** сразу готовая карточка, без конструктора на входе. Пустые поля
не показываются. «Настроить» — необязательный второй шаг. Настройки запоминаются.

**Пометка приложения:** встроена в композицию, с ником пользователя. НЕ снимается
за деньги.

**Фото пластинки:** в MVP, офлайн. Облачное хранение — позже, с бэкендом.

---

## 7. Автопредложения коллекций

При накоплении паттерна (≥ N пластинок) — карточка-предложение. Редко, по делу,
один тап. Считается локально.

MVP: жанр, исполнитель, десятилетие, недавние приобретения, цвет винила.
Позже: лейбл, страна прессинга, незавершённые дискографии.

---

## 8. Монетизация

Модель платная, без рекламы. Ожидания умеренные, план масштабируется.

**Бесплатно навсегда:** каталог любого размера, добавление, теги-коллекции,
вишлист, трекинг затрат, базовая витрина с шарингом, базовые шаблоны.

**Платно — разовые покупки + пакет со скидкой:** премиальные стили витрины;
расширенная статистика. Бандл — обе позиции дешевле суммы. Число позиций малое.

**Позже, с бэкендом:** облачная синхронизация/бэкап, витрина-страница.

**Структура:** разовые покупки + пакет. Подписка — инструмент «серьёзной»
модели, прибережена.

**Убрано из плана:** донаты, реклама.

---

## 9. БЛОК «НА ПОДУМАТЬ»: лимит на размер коллекции

В MVP лимита НЕТ — монетизируем фичами, чтобы не оттолкнуть ядро ниши на старте.
Лимит — резервная гипотеза для теста. До внедрения решить параметры:
- Порог (100 / 150 / 200), цена снятия (~$5 — соотнести с ценой стилей/
  статистики, не должно быть дешевле премиум-стилей), входит ли в бандл,
  тайминг сообщения (предупреждать заранее честнее, чем бить стеной).

## 10. ДИЗАЙН И ВИЗУАЛЬНЫЙ ЯЗЫК (новое в v4)

### Зафиксированное направление: «Аналоговый стол»
Выбрано из 5 концептов после отбора по критериям автора (тёплая винтажная
температура, винил как объект — умеренным акцентом; отвергнуты резкость,
корпоративная стерильность, генерик-минимализм).

Характер: тёмный тёплый фон, золотой акцент, предметная метафора рабочего стола
меломана. Винил как объект подан деликатно (диск с бороздками, грейдинг
состояния), без ретро-китча.

### Визуальный язык
- **Типографика:** Fraunces (serif) — заголовки и «голос»/вайб; IBM Plex Mono —
  служебные подписи, лейблы, метаданные.
- **Палитра:** тёмный тёплый фон, золото как единственный акцент, приглушённые
  цвета обложек.
- **Компоненты:** «felt»-панель (блок проигрывателя сверху главного экрана),
  карточки пластинок, грейдинг состояния (VG/VG+/NM), нав-бар.

### Навигация (зафиксировано)
- **Нижняя панель — 3 вкладки:** Полка (главный экран), Вишлист, Витрина.
- **Профиль** переехал из нижней панели в **верхний угол** главного экрана
  (иконка-аватар). Причина: профиль — редко посещаемый раздел, не заслуживает
  места в нижней панели среди часто используемых.
- **Карта винил-сторов** — иконка рядом с профилем, в верхнем углу. Отдельный
  полноэкранный вид. НЕ внутри вишлиста (разные ментальные категории), НЕ
  отдельная вкладка (редкое действие). Остаётся фичей пост-MVP — место
  зарезервировано, но в первую версию не едет.
- Плавающая кнопка «+» для добавления — пока остаётся (вопрос центральной
  кнопки добавления отложен).

### Проработанные экраны (статичный макет, 10 шт.)
Главный экран, карточка пластинки, добавление «по одной», добавление «списком»,
вишлист, витрина-карточка пластинки, витрина-карточка коллекции, пустой первый
запуск, профиль, карта сторов.

Статус: облик утверждается. Следующий шаг по дизайну — «оживление»
(кликабельный прототип с переходами).

## 11. MVP — итоговый состав

- Мультиколлекции через теги.
- Каталог с автозаполнением через поиск Discogs + ручное редактирование.
- Экран добавления: режимы «по одной» и «списком» с серийным сканированием.
- Трекинг затрат (без рыночного PnL).
- Вишлист.
- Витрина-картинка: карточки пластинки и коллекции (полка + сетка).
- Фото пластинки (офлайн).
- Автопредложения коллекций (5 типов).
- Вайб во всех текстах по принципу дозирования.
- Монетизация: премиальные стили + расширенная статистика + бандл.
- Лимита на размер коллекции НЕТ.
- Дизайн в направлении «Аналоговый стол».

## 12. Дорожная карта (после MVP)

- Витрина-страница + облачное хранение фото + бэкенд — одной волной.
- Карта винил-сторов (OpenStreetMap / Overpass API) + пользовательские

- «Истории пластинки» — необязательное поле.
- Остальные автопредложения.
- Облачная синхронизация/бэкап как платная фича.
- Возможный выход в англоязычные сообщества.
- Развитие в сторону универсального коллекционера.

## 13. Раскрутка

**Аудитория первой волны:** Сербия + русскоязычное сообщество в Сербии — как
тестовый полигон.

**Принцип:** заходить как участник, не как промоутер. Распространение — через
демонстрацию ценности и витрины, которые пользователи постят сами.

**Этапы:** (0) до релиза — 5–10 знакомых коллекционеров тестируют, автор латает
очевидное; (1) запуск в одном сообществе; (2) со сдвигом — второе сообщество;
дальше — осторожный выход в англоязычные среды.

---

## 14. Вычеркнуто (и почему)

- **NFT-витрина** — отторжение в нише, несуществующая боль, тяжёлый бэкенд.
- **Внутренний маркетплейс** — тяжёлый бэкенд, платежи, модерация, конкуренция.
- **Музыкальные ивенты рядом** — нет открытых данных, слабая связь с винилом.
- **Каталог приложений для прослушивания** — низкая отдача.
- **Платное снятие пометки** — вредит виральности.
- **Донаты** — символический доход, конфликт с платными фичами.
- **Реклама** — конфликт с вайбом, витриной и характером ниши.

## 15. Открытые вопросы для следующих итераций

- Оживление макета — кликабельный прототип с переходами.
- Центральная кнопка добавления vs плавающий «+» (отложено).

- Конкретные тексты в вайбовом тоне (микрокопия, пустые экраны, юбилеи).
- Технологический стек / выбор инструментов ИИ-разработки.
- Приоритет и порядок разработки фич внутри MVP.
