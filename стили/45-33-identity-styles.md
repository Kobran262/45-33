## Инструкции для ИИ-кодогенератора

**Главное правило.** Используй ТОЛЬКО токены и правила из этого файла. Не
вводи свои цвета, шрифты, размеры. Если для какого-то элемента нет правила —
выбирай ближайшее по смыслу из существующих токенов, а не придумывай новое.

**Как применять.**
1. Скопируй блок CSS-переменных из раздела «Токены» в корень стилей проекта.
2. Все цвета в компонентах задавай через `var(--token-name)`, не хардкодь HEX.
3. Шрифты подключай из Google Fonts (ссылки в разделе «Типографика»).
4. Стили карточек витрины — это **готовые шаблоны**. В UI-выбора стиля
   показывай их в порядке, указанном в файле. Базовые — без метки, PRO — с
   меткой PRO.
5. **Не объединяй** правила оформления приложения и оформления карточек
   витрины. Это разные слои: первое — само приложение, второе — генерируемый
   контент для шаринга наружу.

## РАЗДЕЛ I — АЙДЕНТИКА ПРИЛОЖЕНИЯ

### 1. Логотип

Знак — «переключатель скоростей»: окружность диска, две короткие риски сверху
и снизу (положения тумблера), числа 45 и 24 внутри. Передаёт идею двух
скоростей вращения винила.

**SVG-исходник (использовать как есть):**
```svg
<svg viewBox="0 0 54 54" xmlns="http://www.w3.org/2000/svg">
  <circle cx="27" cy="27" r="24" fill="none" stroke="currentColor" stroke-width="2.4"/>
  <line x1="27" y1="6" x2="27" y2="14" stroke="currentColor" stroke-width="2.4" stroke-linecap="round"/>
  <line x1="27" y1="40" x2="27" y2="48" stroke="currentColor" stroke-width="2.4" stroke-linecap="round"/>
  <text x="27" y="24" font-family="IBM Plex Mono" font-size="9" font-weight="600" fill="currentColor" text-anchor="middle">45</text>
  <text x="27" y="37" font-family="IBM Plex Mono" font-size="9" font-weight="600" fill="currentColor" text-anchor="middle">33</text>
</svg>
```

`fill="currentColor"` позволяет перекрашивать через CSS. На тёмном фоне знак —
золотой, на золотом — тёмный, на кремовом — тёмный.

**Минимальный размер:** 24px. Меньше — текст «45/33» внутри сливается.
Для иконок < 24px использовать упрощённую версию (только круг и риски).

### 2. Иконка приложения (для сторов)

Иконка строится на том же знаке: золотой квадрат с скруглением радиус 22%,
внутри — тёмный знак переключателя без текста (упрощённая версия).

```
Фон:           --gold (#C98F3C)
Знак:          --bg (#211D18)
Скругление:    22% от стороны
Padding знака: 18% от стороны
```

### 3. Палитра — токены

**Базовый набор (тёмная тема, по умолчанию):**

```css
:root {
  /* Фон */
  --bg:           #211D18;  /* основной фон экрана */
  --bg-deep:      #191510;  /* нижняя нав-панель, глубокий фон */
  --panel:        #2D2820;  /* карточки, блоки */
  --panel-line:   #3D362B;  /* границы карточек */
  --row-line:     #322C23;  /* разделители строк */

  /* Чернила (текст) */
  --ink:          #EAD9B6;  /* основной заголовочный текст */
  --ink-soft:     #D8CFBF;  /* обычный текст */
  --ink-mut:      #9A8F78;  /* приглушённый текст */
  --ink-faint:    #8A7F6A;  /* служебные подписи, метки */

  /* Акцент — единственный цветной */
  --gold:         #C98F3C;  /* акцент, CTA, активные состояния */
  --gold-soft:    #B88A4A;  /* вторичный акцент */

  /* Семантика */
  --ok:           #7FA86A;  /* успех, грейдинг NM/VG+ */
  --warn:         #C97A5A;  /* предупреждения, дубликаты */
}
```

**Правила использования цвета:**
- Золото — **единственный** цветной акцент. Не вводи синий, зелёный, фиолетовый
  как «второстепенные акценты». Семантические `--ok` и `--warn` — только для
  статусов (грейдинг, предупреждения), не для дизайна.
- На одном экране золото занимает не больше **15% площади**. Иначе теряет
  акцентную роль.
- CTA-кнопки (главное действие) — золотом. Вторичные действия — `--ink-mut`
  или прозрачным фоном с рамкой `--panel-line`.
- Текст: заголовки `--ink`, обычный текст `--ink-soft`, второстепенный
  `--ink-mut`, мелкие подписи `--ink-faint`. Не используй чисто белый (#FFF).

### 4. Типографика

Подключение:
```html
<link href="https://fonts.googleapis.com/css2?family=Fraunces:ital,opsz,wght@0,9..144,400..600;1,9..144,400&family=IBM+Plex+Mono:wght@400;500;600&family=Archivo:wght@400..600&display=swap" rel="stylesheet">
```

**Три шрифта, у каждого своя роль:**

| Шрифт            | Роль                                          | Где использовать                          |
|------------------|-----------------------------------------------|-------------------------------------------|
| **Fraunces**     | Заголовки и «голос» (вайбовые фразы курсивом) | H1/H2, названия пластинок, цитаты         |
| **IBM Plex Mono**| Служебные подписи, метки, числа               | Лейблы полей, мета, грейдинг, year-числа  |
| **Archivo**      | Обычный текст UI                              | Параграфы, описания, базовый текст        |

**Шкала размеров:**

```
H1 (главные заголовки экрана):    Fraunces 600, 23–26px, line-height 1.05
H2 (подзаголовки):                Fraunces 600, 18–21px, line-height 1.1
H3 (рубрики):                     IBM Plex Mono 500, 10px, letter-spacing 2px, UPPERCASE
Текст основной:                   Archivo 400, 13–14px, line-height 1.5
Подписи / меты:                   Archivo 400 или Plex Mono 400, 10.5–11px
Очень мелкое (служебное):         Plex Mono 400, 9–10px, letter-spacing 1.5px
```

**Курсив Fraunces** — для вайбовых фраз (цитаты, истории пластинок, тёплые
наблюдения). Не использовать для обычного текста.

### 5. Сетка и отступы

Базовая единица — **4px**. Все отступы и размеры — кратные 4.

**Стандартные отступы:**
- Внешний боковой паддинг экрана: **22px**
- Внутренний паддинг карточки: **13–16px**
- Между строками в списке: **10px** (paddingTop/Bottom)
- Между секциями экрана: **16–22px**
- Между связанными элементами внутри блока: **6–10px**

**Радиусы скругления:**
- Большие блоки (felt-панель): **16px**
- Карточки в списках: **12px**
- Кнопки: **10–11px**
- Поля ввода: **9–11px**
- Чипы / теги: **20px** (овальные)
- Маленькие метки (PRO, grade): **3–5px**

### 6. Базовые компоненты

#### Кнопка primary (золотая, CTA)
```css
background: var(--gold);
color: var(--bg);
font-family: 'IBM Plex Mono', monospace;
font-size: 11px;
font-weight: 500;
letter-spacing: 1px;
text-transform: uppercase;
padding: 13px;
border-radius: 10px;
```

#### Кнопка ghost (вторичная)
```css
background: transparent;
color: var(--ink-mut);
border: 1px solid var(--panel-line);
/* остальное — как у primary */
```

#### Поле ввода
```css
background: var(--panel);
border: 1px solid var(--panel-line);
border-radius: 9px;
padding: 11px 13px;
font-size: 13px;
color: var(--ink);
```
Лейбл поля — `IBM Plex Mono 9px, letter-spacing 1.5px, UPPERCASE, --ink-faint`.

#### Чип / тег
```css
font-family: 'IBM Plex Mono', monospace;
font-size: 10px;
padding: 6px 12px;
border-radius: 20px;
border: 1px solid var(--panel-line);
color: var(--ink-mut);
/* активный: */
background: var(--gold); color: var(--bg); border-color: var(--gold); font-weight: 500;
```

#### Карточка списка пластинок (строка)
```css
display: flex; gap: 12px; align-items: center;
padding: 10px 0;
border-bottom: 1px solid var(--row-line);
/* обложка: 44×44px, border-radius 3px, box-shadow 0 3px 9px rgba(0,0,0,0.45) */
/* название: Fraunces 600, 14px, --ink */
/* артист и год: Archivo, 10.5px, --ink-faint */
```

#### Грейдинг состояния
```css
font-family: 'IBM Plex Mono', monospace;
font-size: 10px;
color: var(--ok);
border: 1px solid #3D4A30;
padding: 2px 6px;
border-radius: 3px;
```
Значения: `M`, `NM`, `VG+`, `VG`, `G+`, `G`, `F`, `P` — стандарт Goldmine.

### 7. Состояния и интерактив

- **Тап / hover**: лёгкое затемнение через `opacity: 0.7` на 100мс или
  `background: var(--panel-line)` для строк списка.
- **Активная вкладка / выбранный элемент**: золотом (текст или фон).
- **Disabled**: `opacity: 0.4`, курсор `not-allowed`.
- **Загрузка**: спиннер на `--gold`, по возможности — крутящаяся пластинка
  как лоадер.

### 8. Что НЕ делать

- Не вводи градиенты (кроме CSS-имитации бороздок пластинки).
- Не используй тени с цветным оттенком — только чёрные.
- Не делай скруглений > 22px на блоках (кроме круглых элементов).
- Не используй стандартный системный синий (`#007AFF`, `#0A84FF`) — он
  ломает палитру.
- Не используй эмодзи в UI как иконки — только в исключительных
  случаях для вайба (например, «💡 подсказка» в автопредложениях).
- Не вводи рамок шире 2.5px нигде, кроме главной CTA-кнопки.

## РАЗДЕЛ II — СТИЛИ КАРТОЧЕК ВИТРИНЫ

### Общая структура карточки витрины

**Карточка пластинки** содержит:
1. Обложка пластинки (центральный элемент)
2. Артист и название
3. Метаданные (год, прессинг, лейбл)
4. Опционально — строка-история
5. Пометка снизу: ник пользователя + лого «45/33»

**Карточка коллекции** содержит:
1. Заголовок коллекции
2. Сетка обложек (или формат «полка»)
3. Краткая статистика (3 числа)
4. Пометка снизу: ник + лого «45/33»

Пометка снизу — **обязательный** элемент любой карточки. Не убирается за
деньги. Подача — как «авторская подпись», не как наклейка. Содержит ник
пользователя и название приложения в типографике карточки.

### Шесть стилей

Базовые (2) показываются всем пользователям. Премиальные (4) — за деньги, с
меткой `PRO` в селекторе. Порядок отображения в UI:
1. → 2. → 3. (PRO) → 4. (PRO) → 5. (PRO) → 6. (PRO)

---

### Стиль 1. «Стол» — БАЗОВЫЙ (по умолчанию)

Стиль приложения. Тёмный тёплый фон, золотой акцент. Совпадает с эстетикой
самого UI — это «дефолт», когда пользователь хочет «как в приложении».

**Токены:**
```
background:    #1A1712
border:        1px solid #3D362B
heading color: --ink (#EAD9B6)
muted color:   --ink-faint (#8A7F6A)
accent:        --gold (#C98F3C)
heading font:  Fraunces 600
meta font:     IBM Plex Mono 500
```

**Особенности:** обложка по центру, лёгкая тень. Метаданные в три колонки
(год / прессинг / лейбл) с подписями `UPPERCASE`. История курсивом
Fraunces, центрирована.

---

### Стиль 2. «Бумага» — БАЗОВЫЙ

Светлый, кремовый. Тон газетной вкладки или конверта пластинки. Альтернатива
тёмному для тех, кто предпочитает светлое.

**Токены:**
```
background:    #ECE7DD
border:        1px solid #B8B1A0
heading color: #232019
muted color:   #86806F
accent:        #B04A2A  /* терракот вместо золота — тёплый и читаемый */
heading font:  Fraunces 600
meta font:     IBM Plex Mono 500
```

**Особенности:** контрастная тёмная типографика на кремовом, обложка с тонкой
тёмной обводкой 1px вместо тени. Метаданные в строку через `·`.

---

### Стиль 3. «Журнал» — PRO

Стиль страницы музыкального журнала. Жирная линейка сверху и снизу, заголовок
крупный, обложка крупная слева, текст справа в две колонки. Подходит для
карточки одной пластинки лучше, чем для коллекции.

**Токены:**
```
background:    #ECE7DD
heading divider: 2px solid #232019  (сверху и снизу карточки)
heading color: #232019
muted color:   #86806F
kicker color:  #B04A2A  (мелкая красная плашка-рубрика сверху)
heading font:  Instrument Serif 400  (новый шрифт для этого стиля)
meta font:     IBM Plex Mono 500
```

**Подключить дополнительно:**
```
Instrument Serif (Google Fonts)
```

**Особенности:** «рубрика» сверху мелкими красными буквами («★ ИЗ КОЛЛЕКЦИИ»),
заголовок Instrument Serif в стиле журнального заголовка, история курсивом
крупно — это «лид-абзац».

---

### Стиль 4. «Неон» — PRO

Тёмный, но с холодным электрическим акцентом. Глубокий синий фон, ярко-розовая
типографика. Эстетика виниловой обложки 80-х / синтвейв. Контрастирует с тёплой
основой приложения — это сознательно для тех, кому хочется громко.

**Токены:**
```
background:    #1A1530
border:        1px solid #2E2855
heading color: #F5E1FF
muted color:   #8A82A5
accent:        #FF4D8F  /* розовый-неон */
secondary:     #4DD8FF  /* голубой-неон, для меток */
heading font:  Fraunces 600
meta font:     IBM Plex Mono 500
```

**Особенности:** обложка с лёгким glow (box-shadow 0 0 30px rgba(255,77,143,0.3)).
Метки — голубым, акцентный текст — розовым. Не использовать золото вообще.

---

### Стиль 5. «Минимал-моно» — PRO

Чёрно-белый, типографический. Никакого цвета, никакой обложки в полную
ширину — обложка в чёрно-белом фильтре (grayscale 1). Для тех, кто любит
строгую графику.

**Токены:**
```
background:    #F4F4F4
border:        1px solid #1A1A1A
heading color: #1A1A1A
muted color:   #6B6B6B
accent:        #1A1A1A  /* нет цветного акцента, всё контрастом */
heading font:  Bricolage Grotesque 700
meta font:     IBM Plex Mono 500
```

**Подключить дополнительно:**
```
Bricolage Grotesque (Google Fonts)
```

**Особенности:** обложка применить filter: grayscale(1) contrast(1.1).
Заголовок крупный, прижатый влево, гротеск. Никаких декоративных элементов —
только сетка, типографика и обложка-в-ч/б.

---

### Стиль 6. «Поляроид» — PRO

Обложка как фото на поляроиде: белая рамка снизу шире, чем по бокам. Стиль
«сфоткал свою пластинку и показываю». Лёгкий наклон карточки на 1–2 градуса.
Скевоморфизм, но дозированный.

**Токены:**
```
background:    #FBFAF5     /* очень светлый кремовый */
photo-border:  #FFFFFF     /* белая рамка вокруг обложки */
border-bottom-extra: 48px  /* нижняя часть рамки шире */
shadow:        0 18px 36px rgba(0,0,0,0.2)
heading color: #2A2218
muted color:   #8A8170
heading font:  Fraunces 600
meta font:     IBM Plex Mono 500
caption font:  'Caveat' 500  /* рукописный — только для подписи под фото */
rotation:      -1.5deg      /* лёгкий наклон */
```

**Подключить дополнительно:**
```
Caveat (Google Fonts)  — для рукописной подписи
```

**Особенности:** обложка обрамлена белой «поляроид-рамкой», заголовок крупно
под рамкой, опциональная рукописная подпись Caveat курсивом (как будто
владелец подписал фото от руки). Лёгкий поворот всей карточки.

### Правила, общие для всех стилей

1. **Структура полей одинакова** — меняется только оформление. Артист, название,
   год, прессинг, история, пометка — везде. Это позволяет переключать стиль без
   потери данных.
2. **Пометка с ником пользователя и названием «45/33»** — обязательна в каждом
   стиле. Шрифт и расположение могут варьироваться, но **сам факт пометки —
   нет**.
3. **Цена покупки никогда не выносится** ни в одном стиле. Это приватное.
4. **Пустые поля не показываются** в карточке. Если пользователь не заполнил
   прессинг — не должно быть строки «Прессинг: —». Просто пропускаем.
5. **Соотношение сторон карточки** — 4:5 (вертикальная) для соцсетей. Это
   стандарт Instagram-постов и сторис.

### Какой стиль для какой карточки

Большинство стилей одинаково работают и для карточки пластинки, и для
коллекции, но есть нюансы:

| Стиль        | Пластинка | Коллекция | Примечание                          |
|--------------|-----------|-----------|-------------------------------------|
| Стол         | ✓         | ✓         | универсальный                       |
| Бумага       | ✓         | ✓         | универсальный                       |
| Журнал       | ✓         | ⚠         | для коллекции слабее, нужна адаптация |
| Неон         | ✓         | ✓         | универсальный                       |
| Минимал-моно | ✓         | ✓         | универсальный                       |
| Поляроид     | ✓         | ⚠         | коллекция как «стопка поляроидов» — отдельная задача |

«⚠» означает: стиль можно сделать и для коллекции, но требует доработки.
В первой версии можно ограничить эти два стиля только карточкой одной пластинки.

## РАЗДЕЛ III — НАБОР PRO КАК ПРОДУКТ

**Что покупает пользователь:** доступ ко всем PRO-стилям пакетом (стили 3–6).
Не поштучно. Это упрощает магазин и не превращает выбор стиля в торговлю
поштучно.

**Расширение со временем:** новые PRO-стили добавляются в этот же набор.
Пользователь, купивший PRO один раз, получает доступ к новым стилям
автоматически. Это и есть «бесконечный источник» монетизации из концепта —
не «купи новый стиль за €1», а «PRO открывает всё, и оно растёт».

**Как показывать PRO в селекторе стилей:**
- В UI выбора стиля показывай все 6 квадратиков подряд.
- На PRO-стилях — маленькая золотая плашка `PRO` в верхнем правом углу
  квадратика.
- При тапе на PRO-стиль без покупки — ведёт на экран магазина (Crate Premium
  / 45/33 Premium), не блокирует молча.
